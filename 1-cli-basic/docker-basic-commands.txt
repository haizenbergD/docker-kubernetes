# Run container from image

1. `docker run <image name> <default command override>`
Examples: 
- 'docker run busybox echo hi'
- 'docker run busybox ls'

2. `docker ps` - list of the running containers
`docker ps --all` - list all running and stopped containers

3. `docker create <image-name>` - create and not start container

4. `docker start -a {container_id}` - start a container
'a' flag - print out container output

5. Restart container that was stopped
docker start -a {container_id}

6. Clean up containers
`docker system purne`

7. Get container logs
`docker logs {container_id}`

8. Stop container
`docker stop ${container_id}` - send stop signal to process, will not stop immediately ( it try to stop it in 10 seconds and run kill after )
`docker kill ${container_id}` - send kill signal to process,  not stop immediately

9. Docker exec
Execute addinional command inside the container.

docker exec -it <container_id> <command>

-it - allow us to provide input to the container (connect inside)
where -i - connect to container terminal, -t - format text

10. Connect to container system (full terminal access)
docker exec -it {container_id} sh
